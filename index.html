CTF（Capture The Flag，夺旗赛）起源于 1996 年 DEFCON 全球黑客大会，是网络安全爱好者之间的竞技游戏。

Pwn - 二进制漏洞利用

Pwn 题目主要考察二进制漏洞的发掘和利用，需要对计算机操作系统底层有一定的了解。在 CTF 竞赛中，PWN 题目主要出现在 Linux 平台上。
ctfer 通过对远程题目服务器上挂载的题目进行分析，寻找漏洞从而利用，最终获取到被攻击端 shell。

Linux Pwn

大多数 PWN 题目都是在 Linux 平台上，所以 Linux pwn 最为重要。你需要学习非常多的漏洞知识，这里提供一个学习链接：CTF Wiki 圈内最好系统学习网站。

栈溢出漏洞：
程序向栈中某个变量中写入的字节数超过了这个变量本身所申请的字节数，因而导致与其相邻的栈中的变量的值被改变

格式化字符串漏洞：
printf("Color %s"); 没有提供参数第二个参数的 printf 会将栈上存储格式化字符串地址的变量进行解析，此例子会解析其地址对应的字符串

堆溢出漏洞：
程序向某个堆块中写入的字节数超过了堆块本身可使用的字节数，因而导致了数据溢出，并覆盖到物理相邻的高地址的下一个堆块

整数溢出漏洞：
程序中的数据超过其数据类型的范围，则会造成溢出，整数类型的溢出被称为整数溢出

沙盒逃逸：
我们通常所说的 Python 沙箱逃逸就是绕过模拟的 Python 终端，最终实现命令执行

内核（Kernel）漏洞：
kernel 也是一个程序，用来管理软件发出的数据 I/O 要求，将这些要求转义为指令，交给 CPU 和计算机中的其他组件处理，kernel 是现代操作系统最基本的部分

Windows Pwn

Windows Pwn 会相对于 Linux Pwn 来说难度会较高些，在传参、函数调用、库函数对比、内核上也会有很大不同。所以当 PWN 能力有提高后，可以开始接触 Windows Pwn 有关的题目。
